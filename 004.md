## Exercice 14 — Protection de `test_sec` pour le groupe

Appliquez les bons droits sur `test_sec` et ses fichiers afin de permettre aux membres de votre groupe de :

| Objectif                                               | Droits nécessaires                     | Commande                                              |
| ------------------------------------------------------ | -------------------------------------- | ----------------------------------------------------- |
| Lire le contenu des fichiers                           | dossier --x \| fichier r--             | `chmod g=x test_sec && chmod g=r test_sec/file{1..5}` |
| Lister les fichiers sans pouvoir lire leur contenu     | dossier r-- \| fichier ---             | `chmod g=r test_sec && chmod g= test_sec/file{1..5}`  |
| Lire un fichier connu sans pouvoir lister les fichiers | dossier --x \| fichier r--             | `chmod g=x test_sec && chmod g=r test_sec/file{1..5}` |
| Modifier un fichier existant                           | dossier --x \| fichier -w-             | `chmod g=x test_sec && chmod g=w test_sec/file{1..5}` |
| Ajouter et effacer un fichier                          | dossier -wx \| fichier (ce qu'on veut) | `chmod g=wx test_sec && chmod g= test_sec/file{1..5}` |

**Commande(s) complètes :**
```bash
# 1. Lire le contenu des fichiers
chmod g=x test_sec && chmod g=r test_sec/file{1..5}

# 2. Lister les fichiers sans pouvoir lire leur contenu
chmod g=r test_sec && chmod g= test_sec/file{1..5}

# 3. Lire un fichier connu sans pouvoir lister les fichiers
chmod g=x test_sec && chmod g=r test_sec/file{1..5}

# 4. Modifier un fichier existant
chmod g=x test_sec && chmod g=w test_sec/file{1..5}

# 5. Ajouter et effacer un fichier
chmod g=wx test_sec && chmod g= test_sec/file{1..5}
```

## Application des permissions — Identifier les droits d'accès

Appliquez les droits sur l'arborescence ci-dessous selon les règles :
- Les fichiers `.txt` ne sont **pas exécutables**
- Le propriétaire `pier` possède **tous les droits** sur toute l'arborescence
- Les utilisateurs du **même groupe** peuvent uniquement :
  - **Lire** le contenu des dossiers et fichiers de `./d1`
  - **Ajouter, modifier et lire** dans `./d2`
- Les **autres utilisateurs** n'ont **aucun droit**

```
home [dr-xr-xr-x]
└── pier [rwx---x---]
    ├── d1 [rwxr-x---]
    │   ├── f1.txt [rw-r-----]
    │   ├── f2.txt [rw-r-----]
    │   ├── f3.txt [rw-r-----]
    │   ├── f4.txt [rw-r-----]
    │   └── f5.txt [rw-r-----]
    └── d2 [rwxrwx---]
        ├── f1.txt [rw-rw----]
        ├── f2.txt [rw-rw----]
        ├── f3.txt [rw-rw----]
        ├── f4.txt [rw-rw----]
        └── f5.txt [rw-rw----]
```

**Commande(s) pour appliquer les droits :**
```bash
chmod u=rwx,g=x,o= ~/pier
chmod u=rwx,g=rx,o= ~/pier/d1
chmod u=rw,g=r,o= ~/pier/d1/f{1..5}.txt
chmod u=rwx,g=rwx,o= ~/pier/d2
chmod u=rw,g=rw,o= ~/pier/d2/f{1..5}.txt
```
